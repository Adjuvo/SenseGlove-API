Detailed Description

A device of which you can directly control the amplitude of vibration motors on each finger, at a set frequency. Note: You'll still have to turn these off!

Set the vibration amplitude of a particular finger to a specific level

Parameters
amplitudeValue will be clamped between [0...1], where 0.0 means no vibration, and 1.0 means full vibration.
fingerThe finger to which to send the command.

Devices that 'only' have on/off vibration control will treat any value > 0.0 as 1.0.

Returns
Returns true if the command was sucessfully added to the queue. If false, the device might not support the chosen finger(s) or have the appropriate firmware.

bool SGCore.Haptics.IFingerVibro_SetLevels.QueueVibroLevels ( float[]  amplitudes)

Queue a list of vibration amplitudes, between 0.0 and 1.0. Your list should be sorted from thumb to pinky.

Parameters
amplitudeArray containing the vibration amplitudes, from 0.0 (no vibration) to 1.0 (full vibration). A value < 0.0f means this finger won't be updated.

Devices that 'only' have on/off vibration control will treat any value > 0.0 as 1.0.

Returns
Returns true if the command was sucessfully added to the queue. If false, the device might not support the chosen finger(s) or have the appropriate firmware.
